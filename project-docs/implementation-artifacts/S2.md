## Acceptance Criteria

### AC1: Validate state doc matches manifest
```
Given a DAG run left a state doc at _bmad-output/dag-automator/orchestration.md
And a dag-manifest.yaml exists alongside it
When the user runs `dag_scheduler.py validate-dag`
Then it checks the state doc's completed_nodes count matches the manifest node list
And reports PASS or FAIL with details
```

### AC2: Detect orphaned nodes
```
Given a prior run was aborted
And the state doc has completed_nodes that reference nodes not in the current manifest
When `dag_scheduler.py validate-dag` runs
Then it reports orphaned nodes with their IDs
```

### AC3: Verify all nodes exist in graph
```
Given a manifest references node IDs
When `dag_scheduler.py validate-dag` runs
Then it checks every referenced node exists in the DAG graph
And reports any missing node IDs
```

### AC4: Validate level boundary consistency
```
Given a paused run at level 2
And the state doc shows levels 0-1 completed
When `dag_scheduler.py validate-dag` runs
Then it confirms that all upstream levels (0, 1) are fully completed
And no downstream level (2+) has incomplete predecessor nodes
```

### AC5: Report as structured output
```
Given `dag_scheduler.py validate-dag` runs
Then it prints a structured report with:
  - Overall verdict: PASS / FAIL / WARN
  - Per-check results with status
  - Any errors with node IDs and context
  - Exit code 0 for PASS, 1 for FAIL, 2 for WARN
```

## Implementation Notes

### New file: `dag_validator.py`

Create a standalone module that performs DAG consistency checks:

```python
class DagValidator:
    """Validates consistency between DAG manifest, state doc, and graph."""

    def validate_manifest_vs_state(self, manifest_path, state_path) -> dict
    def validate_orphaned_nodes(self, manifest, state) -> list
    def validate_node_existence(self, manifest, graph) -> list
    def validate_level_boundaries(self, manifest, state) -> list
    def validate_all(self, manifest_path, state_path, graph) -> dict
```

Each check returns a dict with:
  - `check`: str — check name
  - `status`: "pass" | "fail" | "warn"
  - `details`: list[str] — specific findings
  - `errors`: list[str] — actionable error messages

`validate_all()` returns a report dict:
  - `verdict`: "PASS" | "FAIL" | "WARN"
  - `checks`: list[check_result]
  - `summary`: str — human-readable summary

### Changes to `dag_scheduler.py`

Add a `validate-dag` subcommand (distinct from the existing `validate` which validates story YAML). This new subcommand loads a prior run's artifacts and runs all consistency checks.

```bash
python dag_scheduler.py validate-dag --project /path/to/project
```

### Changes to `state_doc.py`

No changes needed — `DagStateDoc.load()` already exists from S1.
