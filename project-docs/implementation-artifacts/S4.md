> **Sprint:** Phase 5: Production Hardening

> **Status:** pending

---

## Acceptance Criteria

### AC1: Serial vs parallel timing comparison
```
Given a set of independent stories (can run in parallel)
When the benchmark runs the DAG scheduler
Then it measures total wall-clock time
And compares it against estimated serial execution time
And reports speedup ratio
```

### AC2: Agent utilisation per level
```
Given a DAG with varying level widths
When the benchmark runs
Then it records pool size per level
And reports utilisation % (actual agents / max available)
```

### AC3: DAG shape coverage
```
Given the benchmark script
When it runs its suite
Then it tests at least these DAG shapes:
  - Serial chain (10 levels × 1 node)
  - Complete parallel (1 level × 10 nodes)
  - Fan-out/fan-in (3 levels, max width 5)
  - Mixed (5 levels, varying widths)
```

### AC4: Report output
```
Given a benchmark run completes
When it finishes
Then it prints a table with:
  - DAG shape name
  - Total nodes
  - Levels
  - Max width
  - Serial estimate
  - Scheduled time
  - Speedup
  - Avg utilisation
```

---

## Implementation

### benchmarks/bench_dag.py
A standalone benchmark script that:

1. Defines DAG shapes as reusable fixtures
2. Runs each shape through `DagGraph.from_story_list()` + `schedule()`
3. Times scheduling, serial estimate, level-by-level simulation
4. Prints a formatted results table
