## Acceptance Criteria

### AC1: tmux unavailable
```
Given the system has no tmux installed
When the agent pool tries to spawn a session
Then it fails gracefully with a clear error message
And the orchestrator reports the failure without crashing
```

### AC2: Empty story list
```
Given an empty story list
When dag_scheduler.py orchestrate runs
Then it prints a clear message: no stories to process
And exits with a non-zero code
```

### AC3: Single-story sprint
```
Given a single story with no dependencies
When the DAG scheduler processes it
Then it creates one level with one node
And executes it without any parallelism overhead
```

### AC4: Deep DAG (50+ levels)
```
Given a DAG with 50+ levels in a serial chain
When the scheduler processes it
Then it correctly assigns 50+ topological levels
And the critical path spans all nodes
```

### AC5: Graceful shutdown on interrupt
```
Given a DAG run is in progress
When the user sends SIGINT (Ctrl+C)
Then active sessions are cleaned up
And the state doc is saved with PAUSED status
And the run can be resumed later
```

---

## Implementation

### dag_orchestrator.py changes
- Empty story list check at the start of `run_dag()` — early return with clear message
- SIGINT handler that saves state doc and marks PAUSED before exit

### agent_pool.py changes
- tmux availability check in `spawn_node()` — detect and fail gracefully

### artifact_bridge.py changes
- No changes expected — edge cases already handled by git error propagation
