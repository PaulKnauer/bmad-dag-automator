# Story: S1 — Resume from any DAG level

> **Sprint:** Phase 5: Production Hardening
> **Dependencies:** None
> **File scope:** `dag_orchestrator.py`, `state_doc.py`
> **Status:** pending

---

## Acceptance Criteria

### AC1: Detect existing state doc on run
```
Given a prior DAG run left a state doc at _bmad-output/dag-automator/orchestration.md
When dag_orchestrator.run_dag() is called with --resume
Then it reads the existing state doc
And detects the last completed level
```

### AC2: Skip completed levels
```
Given a prior run completed levels 0-1 and paused at level 2
When the orchestrator resumes
Then levels 0-1 are skipped
And execution starts from level 2
```

### AC3: Validate DAG manifest consistency on resume
```
Given a prior run with dag-manifest.yaml
When the orchestrator resumes
Then it validates the current story list matches the manifest
And aborts with a clear error if stories have changed
```

### AC4: State doc tracking during resume
```
Given a resumed run completes levels 2-3
When the run finishes
Then the state doc shows all 4 levels as completed
And the total timeline reflects both the original and resumed segment
```

### AC5: Resume from paused state
```
Given a run is paused due to merge conflict
When the orchestrator resumes with fixed conflict
Then it picks up from the paused level
And the state transitions from PAUSED back to RUNNING
```

## Implementation Notes

### Changes needed in `state_doc.py`:

1. **`DagStateDoc.load(filepath: str) -> DagStateDoc`** — static method to parse an existing state doc from its YAML frontmatter. Parse the `---` frontmatter block with `yaml.safe_load`, reconstruct a `DagStateDoc` instance from its fields.

2. **`DagStateDoc.mark_resumed(level: int, reason: str = "")`** — set status back to RUNNING from PAUSED, add a resumed step entry.

### Changes needed in `dag_orchestrator.py`:

1. **`DagOrchestrator.__init__`** — add `resume: bool = False` parameter and a `_resume_from_level: Optional[int]` field.

2. **`DagOrchestrator.run_dag()`** — before the level loop, if `resume=True`:
   - Load existing state doc from `self.output_dir/orchestration.md`
   - Load existing manifest from `self.output_dir/dag-manifest.yaml`
   - Verify consistency (same nodes, same levels)
   - Set `_resume_from_level` to the last completed level + 1
   - Skip levels ≤ `_resume_from_level` in the loop
   - Print: `⏯️ Resuming from level {N}`

3. **`DagOrchestrator._find_resume_point(state_doc_path: str, manifest_path: str) -> int`** — helper that loads both files, validates consistency, returns the level index to resume from (or -1 if complete, or raises if inconsistent).
