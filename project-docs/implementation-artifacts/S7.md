# Story: S7 — Package as BMAD community module

> **Sprint:** Phase 5: Production Hardening  
> **Dependencies:** S6  
> **File scope:** `_bmad/`  
> **Status:** pending

---

## Acceptance Criteria

### AC1: BMAD module manifest
```
Given the _bmad/ directory
When a BMAD user runs `bmad-help`
Then the DAG Automator is listed as an available tool
And can be invoked via `bmad-dag-schedule`
```

### AC2: Module registered with BMAD
```
Given the module files are in _bmad/
When BMAD's module discovery scans the project
Then the DAG Automator module is found by its manifest
```

### AC3: SKILL.md launcher exists
```
Given a BMAD-compatible IDE (Claude Code, Cursor, etc.)
When the user types `bmad-dag-schedule`
Then it loads the DAG Scheduler skill and presents CLI usage
```

---

## Implementation

### _bmad/bmad-dag-automator/module.yaml
BMAD module manifest with metadata, version, dependencies, and entry points.

### .agents/skills/bmad-dag-schedule/SKILL.md
Skill file that Claude Code / Cursor can load when user types `bmad-dag-schedule`.

### .agents/skills/bmad-dag-validate/SKILL.md
Skill file for `bmad-dag-validate` — validates story sets using the DAG scheduler.

### _bmad/bmad-dag-automator/skills-manifest.csv
Register the new skills so BMAD's `bmad-help` discovers them.
